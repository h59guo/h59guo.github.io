C vs C++: Coding vs debuging


As a CS student in University of Waterloo, I learned C and C++ from CS 136 and CS 246 respectively. They are both powerful and interesting languages that strengthen my coding skills.


C is really a magical language, but you have to do everything on your own. (I think that's why C programmers are usually kind of arrogant XD). Learning C helped me a lot in terms of understanding the memory model (stack, heap, etc), control flow of a program (if statement, all kinds of loops, etc), and most importantly, Abstract Data Type.


ADT is the programming in C that is related to realistic application the most I learned, and I developed a good idea of how they work and how to implement them thanks to CS 136. For assignments in class, I implemented about 10 ADTs myself, all of them with very good feedback: the TA was extremely impressed because they were not only bug free but also well documented.





When I was in an interview, the guy who interviewed me was a Waterloo grad. One thing that he told me was memorable: "Make sure you do well in CS 246, because it tells you what a good programmer should be like."


CS 246 is by far the hardest course that I have encountered. However I did learn a great amount of knowledge from it. It taught me the basic concepts of Object Oriented Languages, many C++ Standard Libraries, and last but not least, useful design patterns. Using Observer pattern, factory pattern and visitor pattern and all the C++ Class knowledge, we built a CC3K RPG game as our final project. check it out on github! It is by far the biggest project I've ever involved, (except for the ones I did while working at Nulogy) and it indeed gave me a sense of achievement.
